HDU-5877-Weak Pair

发表于 2017-03-19 | 分类于 HDU |
字数统计: 1.5k | 阅读时长 ≈ 8

You are given a rooted tree of N nodes, labeled from 1 to N. To the iith node a non-negative value aiai is assigned.An ordered pair of nodes (u,v) is said to be weak if
(1) u is an ancestor of v (Note: In this problem a node u is not considered an ancestor of itself);
(2) au×av≤k.

Can you find the number of weak pairs in the tree?
Input
There are multiple cases in the data set.
The first line of input contains an integer T denoting number of test cases.
For each case, the first line contains two space-separated integers, N and k, respectively.
The second line contains N space-separated integers, denoting a1 to aN.
Each of the subsequent lines contains two space-separated integers defining an edge connecting nodes u and v , where node u is the parent of node v.

Constrains:

1≤N≤105

0≤ai≤109

0≤k≤1018
Output
For each test case, print a single integer on a single line denoting the number of weak pairs in the tree.
Sample Input
1
2 3
1 2
1 2
Sample Output
1

给定一棵树求解满足以下条件的点对个数
1，对于(u,v)，u为v的祖先节点
2，对于(u,v)，有au*av<=k

分析：这题我用了第三种，并且是羊总提供的题解。数列a与数列b的区间和相乘多项式展开就是对应子矩阵的和，那么问题简化为求两个区间和之积为c的倍数（%c==0）的区间和个数。即有多少个(ai+…+aj)(bx+…+by)mod c=0。题解中方法三是求模来算，我们先看看更好理解的不求模的做法。先枚举数列a的区间和，把每个区间和的全部因子找出并统计个数（做法同题解）。接下来枚举b数列的区间和，我们现在要给b数列当前的这个区间和找到若干个与之相乘是c的倍数的a的区间和。由于我们事先已经统计好数列a区间和的因子个数，所以我们只要找到当前b区间和所缺少的c的因子即可。（我们令当前b的区间和为s，s可以拆分成s独有的因子和与c的公因子相乘，我们只要把所缺少的c的因子补全给s，那么就可以凑成c的倍数）。具体的做法就如题解，ans+=a[c/GCD(s,c)]，c/GCD（s,c）即是s所缺少的c的因子。

题解中比这种解法多了一步取模，其实原理基本一样。那为什么取模也能做呢。我们假设某个a数列的区间和为s1，某个b数列的区间和为s2。若要使s1*s2为c的倍数，我们只需要他们分别对c取模的余数之积为c的倍数即可。不妨设s1=c*k1+m1,s1=c*k2+m2,所以s1*s2=k1*k2*c^2+k1*m2*c+k2*m1*c+m1*m2，前三项均为c的倍数，所以只要最后的m1*m2为c的倍数即可，而m1,m2正是s1,s2对c取模的余数。需要注意的是我们取模若是余数为0，即本身被c整除，我们要把a[0]++改成a[c]++,其余做法不变。
