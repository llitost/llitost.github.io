首先我们定义 delta[i] 表示从 i ~ n的数字都增加了delta[i]。 这样的话，对于每次更新区间[a, b]，我们只需要令 delta[a] += d，表示：从a到n每个数都增加了d。。但我们其实只需要从a到b每个数都增加d，所以我再令delta[b + 1] -= d，将之前多增加的减掉，这样的话，就相当于从a到b都增加了d。

查询

我们定义a[i]表示原来数列的第i个数，那么对于每次查询[a, b]的和，就是sum(b) – sum(a – 1)，sum(x)表示数列1 ~ x的和。 我们求1 ~ x时，delta[1]对他的贡献是xdelta[1]，因为delta[1]是表示从1到n都增加delta[1]。 所以sum(x) ＝ a[1] + a[2] + … + a[x] + delta[1] x + delta[2] (x – 1) + delta[3] (x – 2) + … + delta[x] ＝ segma(a[i]) + segma(delta[i] (x – i + 1)) （1 <= i <= x）＝ segma(a[i]) + segma(delta[i]) (x + 1) – segma(delta[i] * i) 这样的话，就是三个前缀和累加和了，所以！！！！轮到树状数组出场。。

正文

题意：有两种操作：

2 ：查询当前集合里最小的由集合里任意元素相加不能得到的值

分析：因为此集合里会有重复元素，那么使用multiset，内部通过平衡二叉树来实现，总是把最小的元素的地址定义为头指针（类似优先队列）。
可以确认的是， 一开始当集合为空集时，SG 值为 1.

对于 x <= SG 值 的情况
因为之前可以组合出 [1, SG 值) 的数， 加入 x 后就可以组合出 [1， SG 值 + x) 的数， 同时原 SG 值更新为 SG 值 + x。
对于 x > SG 值 的情况
因为 x > SG 值， 所以加入 x 对目前的 SG 值没有影响， 但如果以后 SG 值变得比 x 大的话， 就会对其产生影响， 所以先将 x 存起来， 每一次 SG 值得到更新后， 提取出之前存过的数， 看其是否能更新 SG 值。

首先想一下特殊情况，如果只有一个气球，我们要确定高度只能从下往上一层一层地测试，因为如果气球一旦爆了，便无法测出气球的硬度。

如果气球有无数个，那么就可以用二分的方法来确定。

一般地，用d(i, j)表示用i个气球实验j次所能确定的楼层的最大高度。

我们假设第一个气球从第k层扔下，

如果气球爆了，那么剩下的i-1个气球实验j-1次，要能在下面的k-1层确定气球的硬度。所以这个k最大取d(i-1, j-1)+1
气球没爆，那么第1~k层就完全不用管了，i个气球剩下的j-1次测试就直接往上测试就行，最多能测试d(i, j-1)层
所以d(i, j) = d(i-1, j-1) + 1 + d(i, j-1)

然而，这题只有3个气球，所以只需求d(3,j)>=n，j则是答案。数据量很大，需要找到规律:d(1,j)=j,d(2,j)=(jj+j)/2,所以d(3,j)=(jjj+5j)/6;这样就可直接计算。得出以上等式的方法为数学的递推归纳，或是打表。

4

1 51
1 33
2 69
4
0 20523
1 19243
1 3890
0 31492

31492 20523 3890 19243
题目意思：有n个数，进行n次操作，每次操作有两个数pos, ans。pos的意思是把ans放到第pos 位置的后面，pos后面的数就往后推一位。最后输出每个位置的ans。

思路：根据题 目可知，最后插入的位置的数才是最终不变的数，所以可以从最后的输入作第1个放入，依此类推，倒插入。在插入时也有一定的技术，首先创建一棵空线段树时，每个节点记录当前范围内有多少个空位置。在插入时，要注意，一个数放入之后那么这个位置就不用管了，那么树中所有的空位置就是余下的数所对应的位置，也就是把余下的数又可以看作是一个新的集合。那么每次插入都是当前集合的第1次放。

题目链接：http://poj.org/problem?id=1753
题意：一个4*4的棋盘，每个方格都有一个棋子，非黑即白。每次任选一个棋子，实行翻转操作（以当前棋子为中心的十字），当所以棋子都是同种颜色，游戏就结束了。现给定一个初始状态，问完成游戏所需的最小翻转次数，若不存在则输出“Impossible”。

分析：
1.因为每个棋子只有两种状态，我们考虑用01表示，则棋盘的状态可由唯一的16位二进制表示。显然当棋盘为0(全白)或65535(全黑)时，游戏结束。
2.对于一个棋子，翻转偶数次是无意义的，所以每个棋子最多翻转一次。
3.由于0^1=1,1^1=0,所以翻转的操作可通过位运算来操作

解法一：BFS+位压缩。首先求出由全白状态翻转某个棋子得到的基础状态，用于异或操作实行翻转。
例如 0001 0011 0010
1110 于右上角的1进行翻转时，相当于与 0001 进行异或操作，得 0011 ，以此实现翻转的优化。接下来便是bfs搜索每个位置，一旦成功，立即返回答案。
1010 0000 1010
0010 0000 0010
