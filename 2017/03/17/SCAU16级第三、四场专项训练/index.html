通过总结我们可以看出，我们只是要选择奇数a的段数就可以了（偶数天的段数是不会影响到区间的选择的），那么我们首先需要做的是确定边界。
（PS：因为有大佬说我说的不够详细，那我就简单说下思路：因为我们一开始读入数据，可以发现，当我们原本区间是偶数的话，那么加入一个奇数区间，就会影响到最后的答案，而加入一个偶数区间，就不会影响到答案。因此，我们可以先把区间所有的值算出来，然后用奇数的区间来判断，究竟哪些位置是要被加入最后答案的，那么我们就要涉及到边界问题了。）
边界有很多情况，需要特判的有两种：1.我们需要处理不同区间的左右边界重合 2.我们需要处理相同区间的重合 而且注意，在开始加a和结束加a，对区间的影响位置是不一样的。因此，我们需要三个数组（这是我大规模改了三次程序之后找出来的方法） 一个用来储存左边界位置 一个是右边界位置 还有一个是放在最前面，储存左右边界重合（相同区间）的位置。 这样扫过去，我们会发现通过判断前一个值temp加上有没有新进的左边界和有没有新进的重合边界之和是奇数的话，那么这位置也是奇数，这样做下去，我们就能把所有符合条件的当前值加起来，轻松得到答案~~~~

这道题目需要求出子矩阵和%c==0的个数。

  • 方法一：朴素模拟，先预处理出一个矩阵前缀和数组，枚举子矩阵的左上角和右下角，判断%c==0？O(Tn2m2)，显然超时。
  • 方法二：先看一维的情况，预处理出区间前缀和%c数组a（a[0]=1），而要求的子区间数就是前缀和数组a中两两相同的组数。
    例如有数列 3 6 9 要求%2==0的区间，a[0]=0;a[1]=1;a[2]=1;a[3]=0;得结果为2；
扩展到二维的数组，预处理f(i,j,k)表示i到j列的前k行的和模c，然后就是对于每组i,j，统计有多少组两两相同。O(TN2M)
 找相同的两个数的方法可以用一个数组在O（n）的复杂度下做出来。
  • 方法三（正解）：“意思就是问多少个(ai+…+aj)(bx…by)mod c=0。。O(n2)枚举ai,aj，得知他们间一串的和mod c为s(若模为0，则a[c]++，下同)，O(sqrt(s))找s的所有因数xi，a [xi]++。。然后O(m2)枚举bx,by，得和mod c为s，求c与s最小公倍数除以c为S，ans+=a[S]” O(T(n2+m2)sqrt(c))

分析：这题我用了第三种，并且是羊总提供的题解。数列a与数列b的区间和相乘多项式展开就是对应子矩阵的和，那么问题简化为求两个区间和之积为c的倍数（%c==0）的区间和个数。即有多少个(ai+…+aj)(bx+…+by)mod c=0。题解中方法三是求模来算，我们先看看更好理解的不求模的做法。先枚举数列a的区间和，把每个区间和的全部因子找出并统计个数（做法同题解）。接下来枚举b数列的区间和，我们现在要给b数列当前的这个区间和找到若干个与之相乘是c的倍数的a的区间和。由于我们事先已经统计好数列a区间和的因子个数，所以我们只要找到当前b区间和所缺少的c的因子即可。（我们令当前b的区间和为s，s可以拆分成s独有的因子和与c的公因子相乘，我们只要把所缺少的c的因子补全给s，那么就可以凑成c的倍数）。具体的做法就如题解，ans+=a[c/GCD(s,c)]，c/GCD（s,c）即是s所缺少的c的因子。

题解中比这种解法多了一步取模，其实原理基本一样。那为什么取模也能做呢。我们假设某个a数列的区间和为s1，某个b数列的区间和为s2。若要使s1*s2为c的倍数，我们只需要他们分别对c取模的余数之积为c的倍数即可。不妨设s1=c*k1+m1,s1=c*k2+m2,所以s1*s2=k1*k2*c^2+k1*m2*c+k2*m1*c+m1*m2，前三项均为c的倍数，所以只要最后的m1*m2为c的倍数即可，而m1,m2正是s1,s2对c取模的余数。需要注意的是我们取模若是余数为0，即本身被c整除，我们要把a[0]++改成a[c]++,其余做法不变。

解析：来自Toshiba，设d(i)为在总格子数为i，c种颜色，最小长度m条件下染色的方案数。即本题答案为d(n)。
那么对于第i个格子，有两种染法：一种不染，有d(i-1)个方案。
一种是染，由于最小长度是m，那么闭区间[i-m+1，i]都应该染同一种颜色。
接着考虑第i-m个格子，该格子也有染和不染两种情况，如果不染，由于两种颜色应该间隔一个格子，那么有d(i-m-1)c个方案，如果染，那接着考虑第i-m-1个格子，不染的话，有d(i-m-2)c个方案，染的话，接着考虑第i-m-2个格子……
如此下去，到考虑第1个格子时，有染和不染两种情况，都有c个方案，即可得解。
下面分析样例5 3 2：首先，d(1) = 1，d(2) = 1，只能不染。d(3)：考虑第3个格子，如果不染，有d(2)种，如果染，1到3的格子都要染，由于有2种颜色，d(3) = d(2)+c = 3。d(4)：考虑第四个格子，如果不染，有d(3)种，如果染，2到4的格子都要染，在2到4都染情况下，如果1不染，有c种，如果1染，也有c种，d(4) = d(3)+c+c = 7。d(5)：考虑第五个格子，如果不染有d(4)种，如果染，3到5都要染，在这情况下，考虑2，2不染，有d(1)c种，2染，考虑1,如果1染，有c种，不染，有c种，d(5) = d(4)+d(1)c+c+c = 13。

当然如果你算一次d(i)的时间复杂度不是O(1)的话是会超时的，可以用sum(i)保存d(i)+…+d(1)。

I 速配游戏
Time Limit:3000MS Memory Limit:65535K
题型: 编程题 语言: 无限制
描述
有一个速配电视节目。N位男士和N位女士要在摄像机前选出他们合适的伴侣。每位女士按照其对每位男士作为配偶的偏爱程度给每位男士排名次，每位男士也按照其对每位女士作为配偶的偏爱程度给每位女士排名次。这些名次不允许并列。然后每位男士将向心仪的对象求婚，经过“残酷”的竞争之后各自找到适合的伴侣。

最开始的时候每位男士都没有被任何一位女士拒绝。求婚环节会经过很多轮进行，每一轮:

(1)每位男士从还没有拒绝过自己的女士中选出自己认为最理想的一个，并向她求婚。

邮局最近推出了一套纪念邮票，这套邮票共有N张，邮票面值各不相同，按编号顺序为1分，2分，…，N分。

你的任务是求出所有符合要求的方案，以[a, b]的形式输出。

输入只有一行，包含两个数N和M(1≤N， M≤109)。

输出格式

输出不含任何空格。

输入样例
20 15

[4,6]

提交

由题意有：2M=(a+b)(b-a+1),若枚举a或b中任意一个，数据量为1e9，必定超时。固把式子简化，设x=a+b,y=b-a+1,这样，原式成了xy=2M,易知x>=y,所以1<=y<=sqrt(2M),因此遍历y，可得x(记得判断x为整数)，又a=(x-y+1)/2,b=(x+y-1)/2(也要判断整数且a和b在范围内)，符合限制输出答案即可

另一种：对n进行深搜，第一步分解得到n的因子，然后继续对其因子进行分解，直到不能分解为止，例如，对于12，其深搜顺序为12-2-6-2-3-3-4-2，最后输出8，要注意的是，若一个数分解后得到两个相同的因子，则只用对这个因子搜索一次，例如，4=2*2，只用对2搜索一次，题目给的n的范围很大，为防止超时，而数组又不能太那么大，用map来存，搜索一个数前，先判断下a[i]，等于0的话说明还没搜索过，需要搜索，大于0的话直接返回a[i]就行了，这样的话每个数都只用搜索一次了
